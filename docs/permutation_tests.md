# Permutation analyses for ARD enrichment

## Global ARD vs non-ARD test
- Following MR harmonisation, the enrichment module screens the phenotype results to retain only outcomes that passed sensitivity QC and carried at least two instruments, ensuring that the age-related disease (ARD) versus non-ARD comparison rests on robust Mendelian Randomization estimates.【F:R/enrichment_business_logic.R†L313-L320】
- Each surviving phenotype is scored with a precision-weighted, signed evidence metric (sign of the IVW beta multiplied by the -log10 p-value), so that concordant risk elevations and protections are amplified according to statistical support.【F:R/enrichment_business_logic.R†L317-L324】
- The pipeline then permutes the ARD labels either exactly (when combinatorially feasible) or via 10,000 Monte Carlo draws, building a two-sided null distribution for the weighted evidence sum; permutation draws are optionally retained to fuel downstream visualizations.【F:R/enrichment_business_logic.R†L323-L355】【F:R/enrichment_business_logic.R†L84-L205】
- Observed statistics are benchmarked against the permutation ensemble to yield signed enrichment p-values, which are Benjamini–Hochberg adjusted before being stored alongside retained draws for plotting and export within the results bundle.【F:R/enrichment_business_logic.R†L339-L360】
- Run-time orchestration seeds the analysis (seed = 1) and requests retention of up to 10,000 permutation draws, then generates forest-style and vertical violin plots that are written to the exposure’s `enrichment/global` directory via the hierarchical plot saver for figure-ready dissemination.【F:R/run_phenome_mr.R†L530-L626】【F:R/run_phenome_mr.R†L220-L252】

## Cause-level enrichment (GBD levels 1-3)
- For each Global Burden of Disease hierarchy (levels 1–3), the workflow filters to phenotypes annotated with the relevant cause, again enforcing QC pass and instrument count thresholds before computing signed evidence and inverse-variance weights.【F:R/enrichment_business_logic.R†L388-L398】
- In the `cause_vs_rest_all` comparison, the signed evidence within a focal cause is contrasted against the rest of the phenome by permuting the cause membership labels under a global exchangeability assumption, reusing the same two-sided permutation engine with seed offsets to stabilise Monte Carlo variation across causes.【F:R/enrichment_business_logic.R†L400-L471】
- Benjamini–Hochberg correction is applied separately within each cause level so that the resulting q-values highlight cause families where ARD signal enrichment exceeds the null expectation after FDR control.【F:R/enrichment_business_logic.R†L475-L480】
- The orchestrator renders both vertical and forest-oriented violin plots for every cause level, embedding retained permutation draws and observed statistics, and stores the figures under `enrichment/cause_level_{1,2,3}` for reporting alongside the tabular outputs saved in `results.rds`.【F:R/run_phenome_mr.R†L605-L626】【F:R/run_phenome_mr.R†L220-L252】【F:R/run_phenome_mr.R†L956-L963】
